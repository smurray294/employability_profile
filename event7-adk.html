Event 7: Google ADK Hackathon

Date: 23/06/2025

Summary

This event was a Hackathon competition hosted by Google, using their newly released (earlier this year) Agent Development Kit (ADK) platform. Whilst I did not complete a final project for submission, my goal was to just learn about agent development and the platform with some hands-on practice. This is pretty novel technology so making a strong submission in a short time frame was a bit out of the scope of my abilities, but there is a very good chance this technology is a big part of the future so learning about it seemed very valuable. This involved completing a tutorial on creating an agent that can read and summarise the top posts from specific subreddits on www.reddit.com. The tutorial was focussed on gamedev subreddits, but I modified my agent to check for politics and world news events. This taught me basic skills on how to engage with and use the ADK platform, how to interact with various APIs, and gave me some insight on the types of things this technology might be used for going forward.

Additional Comments

The way this platform seems to work is you connect to a well known LLM using their API, provide a set of instructions that define how the agent is meant to operate, and also provide ‘tools’ which it uses to complete those tasks (in this case we defined one tool, which was a function that used reddit’s API to retrieve the top 5 posts on any given subreddit). As it is still a LLM, it is able to provide various other functionality (i.e. generating a tweet, or commenting on political tone, as you can see on the right).

You can run the agent in a local interface and interact with it as you would the usual commercially available LLMs, as seen in the evidence screenshots. Whilst I have only created one agent here, the intended purpose of the Hackathon and the technology in general is to have multiple agents that perform roles and interface to solve or offer a solution to complex problems or workflows.

Event 8: Hugging Face: MCP Course

Date: 23/06/2025

Summary

This course is an introduction to MCP (model context protocol) which is a pretty new technology, introduced last year. The goal of MCP is to provide a consistent standard to allow AI agents to interface with tools, services and data. This makes the lives of users and developers much easier, as it converts the MxN problem (M agents need to be able to interact, ie have different APIs, with N different things) into an M+N problem (M agents connect to a server which connects to N different things, with the server handling the interfacing). It is not widely adopted yet, but does seem to be exceptionally powerful. I worked through the course online and then completed the certification test at the end.

Additional Comments

My main takeaway from the course is that the only challenge or downside to this protocol is adopt-ability - it is all positives for everyone involved, as long as it is widely adopted and used throughout the industry. The rest of the course was mostly introductory, introducing the actors involved (host, client, server), the architecture, and the types of messages and communication between the different actors. Whilst for small scale projects I can see why this might not be necessary, it seems like people in the industry are imagining a world where many different AI agents are communicating with each other and interacting with other information sources online, and in that situation a protocol like this is basically required and has been implemented in similar situations in the past.

Interestingly, the protocol allows the MCP server (who your agent queries when it wants to interact with something) to take the result and send it back to the client and ask for feedback etc. This allows a level of interaction/control from the human user, but it isn’t a stretch to imagine the client being another agent, communicating back and forth, refining the output. Another interesting thing I found was that, whilst historically through Uni it was very important to get the code right, comments were mainly for our own/other human reference, but in the world of agent development, the comments explaining what a function or method does are fundamental - the agents need them to understand the context and functionality of a function.
